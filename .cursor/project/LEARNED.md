# Learned Project Patterns

Use this file for repeated, project-specific patterns discovered while working.
These are not generic Unity or coding rules. They are lessons that help future
work avoid the same mistake or rediscover the same convention.

## When To Add A Learned Pattern

Add an entry only when the pattern is:

- repeated
- project-specific
- high-signal
- useful for future work
- stable enough to guide later tasks

Do not add:

- one-off notes
- temporary debugging observations
- generic Unity advice
- preferences that belong in `CODING_CONVENTIONS.md`
- hard rules that belong in `RULES.md`
